# Quick Start

Let's build a simple counter app with Pyra and React. You'll learn the basics of Pyra's dev server, HMR, and build process.

## Create Your Project

<Terminal>
npm create pyra@latest my-counter-app -- --template react-ts
cd my-counter-app
npm install
</Terminal>

## Start the Dev Server

<Terminal>
npm run dev
</Terminal>

```
  PYRA v1.0.0-beta.1  ready in 52 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h to show help
```

<Callout type="tip">
Notice how fast that was? Pyra starts instantly because it doesn't bundle during development.
</Callout>

## Build Your First Component

Open `src/App.tsx` and replace it with:

<GradientCode language="typescript">
import { useState } from 'react';
import './App.css';

function App() {
  const [count, setCount] = useState(0);

  return (
    <div className="app">
      <h1>🔥 Pyra Counter</h1>
      <div className="card">
        <button onClick={() => setCount(count + 1)}>
          Count is {count}
        </button>
        <p>Edit src/App.tsx and save to test HMR</p>
      </div>
    </div>
  );
}

export default App;
</GradientCode>

## Experience Hot Module Replacement

With the dev server running, try editing the component:

<Step number={1}>
Change the emoji in the `<h1>` tag from 🔥 to ⚡
</Step>

<Step number={2}>
Save the file
</Step>

<Step number={3}>
Watch your browser update **instantly** without losing the counter state
</Step>

<Callout type="note">
This is Pyra's HMR in action. Only the changed component reloads, and React state is preserved.
</Callout>

## Import Assets

Pyra handles static assets automatically. Create `src/assets/logo.svg` or use an existing image:

<GradientCode language="typescript">
import { useState } from 'react';
import logo from './assets/logo.svg';
import './App.css';

function App() {
  const [count, setCount] = useState(0);

  return (
    <div className="app">
      <img src={logo} alt="Logo" width="100" />
      <h1>🔥 Pyra Counter</h1>
      {/* ... rest of component */}
    </div>
  );
}

export default App;
</GradientCode>

<Callout type="tip">
Pyra automatically optimizes images, handles SVGs, and generates proper URLs for all your assets.
</Callout>

## Use Environment Variables

Create a `.env` file in your project root:

```
PYRA_APP_TITLE=My Awesome Counter
PYRA_API_URL=https://api.example.com
```

Access them in your code:

<GradientCode language="typescript">
function App() {
  const appTitle = import.meta.env.PYRA_APP_TITLE;
  
  return (
    <div className="app">
      <h1>{appTitle}</h1>
      {/* ... */}
    </div>
  );
}
</GradientCode>

<Callout type="warn">
Only variables prefixed with `PYRA_` are exposed to your client code. This prevents accidentally leaking secrets.
</Callout>

## Build for Production

When you're ready to deploy:

<Terminal>
npm run build
</Terminal>

```

✓ 34 modules transformed.
dist/index.html                   0.45 kB
dist/assets/index-a3b4c5d6.css    1.23 kB │ gzip: 0.61 kB
dist/assets/index-d4e5f6a7.js    143.21 kB │ gzip: 46.13 kB

✓ built in 1.24s
```

Your optimized app is now in the `dist/` folder, ready to deploy.

The preview server serves your production build exactly as it will appear when deployed.
