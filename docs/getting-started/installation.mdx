# Installation

Get started with Pyra in under a minute. Choose your package manager and let's go.

## Quick Start

<Step number={1}>

### Create a New Project

Use the official scaffolding tool to create a new Pyra project:

<Terminal>
npm create pyra@latest my-pyra-app
</Terminal>

Or with your preferred package manager:

<Terminal>
pnpm create pyra my-pyra-app
</Terminal>

<Terminal>
yarn create pyra my-pyra-app
</Terminal>

</Step>

<Step number={2}>

### Choose a Template

The CLI will prompt you to select a template:

```
? Select a framework: › 
❯ React
  Vue
  Svelte
  Vanilla

? Select a variant: ›
❯ TypeScript
  JavaScript
```

<Callout type="tip">
We recommend starting with **React + TypeScript** for the best developer experience.
</Callout>

</Step>

<Step number={3}>

### Install Dependencies

Navigate to your project and install dependencies:

<Terminal>
cd my-pyra-app
npm install
</Terminal>

</Step>

<Step number={4}>

### Start the Dev Server

<Terminal>
npm run dev
</Terminal>

Your app is now running at `http://localhost:5173` 🔥

</Step>

## Manual Installation

If you prefer to set up Pyra manually in an existing project:

<Step number={1}>

### Install Pyra

<Terminal>
npm install -D pyra
</Terminal>

</Step>

<Step number={2}>

### Create Project Structure

```
my-project/
├── index.html
├── src/
│   ├── main.ts
│   └── app.ts
├── package.json
└── tsconfig.json
```

</Step>

<Step number={3}>

### Add Scripts to package.json

<GradientCode language="json">
{
  "scripts": {
    "dev": "pyra dev",
    "build": "pyra build",
    "preview": "pyra preview"
  }
}
</GradientCode>

</Step>

<Step number={4}>

### Create index.html

<GradientCode language="html">
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Pyra App</title>
  </head>
  <body>
    <div id="app"></div>
    <script type="module" src="/src/main.ts"></script>
  </body>
</html>
</GradientCode>

<Callout type="note">
The `type="module"` attribute is required. Pyra serves your source files as native ES modules.
</Callout>

</Step>

<Step number={5}>

### Create src/main.ts

<GradientCode language="typescript">
import { createApp } from './app';

const app = createApp();
app.mount('#app');

console.log('🔥 Pyra app ignited!');
</GradientCode>

</Step>

<Step number={6}>

### Start Development

<Terminal>
npm run dev
</Terminal>

</Step>

## System Requirements

Pyra requires:

- **Node.js** version 18.0 or higher
- **npm** 7+, **pnpm** 7+, or **yarn** 1.22+

<Callout type="warn">
Pyra uses native ES modules and modern JavaScript features. Node.js 16 and below are not supported.
</Callout>

## TypeScript Configuration

Pyra works with TypeScript out of the box. Here's a recommended `tsconfig.json`:

<GradientCode language="json">
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "allowImportingTsExtensions": true,
    "strict": true,
    "skipLibCheck": true,
    "noEmit": true,
    "jsx": "react-jsx",
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["src"]
}
</GradientCode>

<Callout type="tip">
Pyra automatically reads your `tsconfig.json` and respects your path aliases. No additional configuration needed!
</Callout>

## Verify Installation

Check that Pyra is installed correctly:

<Terminal>
npx pyra --version
</Terminal>

You should see output like:

```
pyra/1.0.0-beta.1
```
