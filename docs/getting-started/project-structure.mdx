# Project Structure

<p className="text-xl text-gray-400 leading-relaxed">
  Learn how Pyra.js organizes your project and what each file does.
</p>

## Default Structure

A typical Pyra.js project looks like this:

<GradientCode language="plaintext">
{`my-app/
├── node_modules/        # Dependencies
├── public/              # Static assets (copied as-is)
│   └── favicon.ico
├── src/                 # Source code
│   ├── assets/          # Processed assets (images, fonts)
│   ├── components/      # React/Vue/Svelte components
│   ├── App.tsx          # Root component
│   ├── App.css          # Component styles
│   └── main.ts          # Application entry point
├── .env                 # Environment variables
├── .gitignore           # Git ignore rules
├── index.html           # HTML entry point
├── package.json         # Project metadata
├── pyra.config.ts       # Pyra configuration
└── tsconfig.json        # TypeScript configuration`}
</GradientCode>

## Key Files

### `index.html`

The HTML entry point. Pyra uses this as the starting point for your app:

<GradientCode language="html" title="index.html">
{`<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/favicon.ico" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Pyra App</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.ts"></script>
  </body>
</html>`}
</GradientCode>

<Callout type="note">
  The `<script type="module">` tag is crucial—it tells Pyra to process this file as an ES module entry point.
</Callout>

### `src/main.ts`

Your application's JavaScript/TypeScript entry point:

<GradientCode language="typescript" title="src/main.ts">
{`import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';
import './index.css';

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);`}
</GradientCode>

### `pyra.config.ts`

Optional configuration file for customizing Pyra's behavior:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';
import react from '@pyra/plugin-react';

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,
    open: true
  },
  build: {
    outDir: 'dist',
    sourcemap: true
  }
});`}
</GradientCode>

### `tsconfig.json`

TypeScript configuration. Pyra respects your TypeScript settings:

<GradientCode language="json" title="tsconfig.json">
{`{
  "compilerOptions": {
    "target": "ES2020",
    "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx",
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}`}
</GradientCode>

## Directory Conventions

### `src/` - Source Code

All your application code lives here. Pyra processes files in this directory:

- **TypeScript/JavaScript** - `.ts`, `.tsx`, `.js`, `.jsx`
- **Styles** - `.css`, `.scss`, `.less`
- **Assets** - `.png`, `.jpg`, `.svg`, `.woff`, etc.

### `public/` - Static Assets

Files in `public/` are copied to the build output as-is:

<GradientCode language="plaintext">
{`public/
├── favicon.ico          # Copied to dist/favicon.ico
├── robots.txt           # Copied to dist/robots.txt
└── images/
    └── logo.png         # Copied to dist/images/logo.png`}
</GradientCode>

Reference public files with absolute paths:

<GradientCode language="html">
{`<img src="/images/logo.png" alt="Logo" />`}
</GradientCode>

<Callout type="tip">
  Use `public/` for files that need specific names (like `robots.txt`) or shouldn't be processed by Pyra.
</Callout>

### `src/assets/` - Processed Assets

Assets in `src/assets/` are processed by Pyra and can be imported:

<GradientCode language="typescript">
{`import logo from './assets/logo.png';

function App() {
  return <img src={logo} alt="Logo" />;
}`}
</GradientCode>

Pyra will:
- Optimize images
- Generate hashed filenames for caching
- Inline small assets as data URLs

## Import Paths

### Relative Imports

<GradientCode language="typescript">
{`import Button from './components/Button';
import { formatDate } from '../utils/date';`}
</GradientCode>

### Absolute Imports (Path Aliases)

Configure in `tsconfig.json`:

<GradientCode language="json">
{`{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"],
      "@components/*": ["./src/components/*"],
      "@utils/*": ["./src/utils/*"]
    }
  }
}`}
</GradientCode>

Then use:

<GradientCode language="typescript">
{`import Button from '@components/Button';
import { formatDate } from '@utils/date';`}
</GradientCode>

## Build Output

Running `pyra build` generates:

<GradientCode language="plaintext">
{`dist/
├── assets/
│   ├── index-a3b4c5d6.js      # Hashed JS bundles
│   ├── index-e7f8g9h0.css     # Hashed CSS bundles
│   └── logo-i1j2k3l4.png      # Hashed assets
├── index.html                  # Entry HTML
└── favicon.ico                 # Static files from public/`}
</GradientCode>

<Callout type="note">
  Hashed filenames enable long-term caching. When files change, the hash changes, busting the cache automatically.
</Callout>

## Organizing Large Projects

For larger applications, consider this structure:

<GradientCode language="plaintext">
{`src/
├── components/          # Reusable UI components
│   ├── Button/
│   │   ├── Button.tsx
│   │   ├── Button.test.tsx
│   │   └── Button.css
│   └── Card/
├── features/            # Feature-based modules
│   ├── auth/
│   │   ├── Login.tsx
│   │   ├── Register.tsx
│   │   └── authSlice.ts
│   └── dashboard/
├── hooks/               # Custom React hooks
├── utils/               # Utility functions
├── types/               # TypeScript types
├── api/                 # API client code
├── assets/              # Images, fonts, etc.
└── styles/              # Global styles`}
</GradientCode>

## Environment-Specific Files

Create environment-specific configs:

<GradientCode language="plaintext">
{`.env                    # Loaded in all cases
.env.local              # Local overrides (gitignored)
.env.development        # Development only
.env.production         # Production only`}
</GradientCode>

Priority (highest to lowest):
1. `.env.[mode].local`
2. `.env.[mode]`
3. `.env.local`
4. `.env`
