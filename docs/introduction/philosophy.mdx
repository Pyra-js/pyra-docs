# Philosophy

<p className="text-xl text-gray-400 leading-relaxed">
  Pyra.js is built on three core principles: <span className="pyra-gradient-text font-semibold">Ignite → Create → Flow</span>
</p>

## 🔥 Ignite

**Start instantly, build immediately.**

Modern development shouldn't require extensive setup or configuration. Pyra ignites your workflow by:

- **Zero-config defaults** - Sensible defaults that work for 90% of projects
- **Instant server startup** - Native ESM means no bundling during development
- **Fast installation** - Minimal dependencies, maximum speed
- **Quick scaffolding** - Create new projects in seconds

<Callout type="tip">
  The best build tool is the one you don't have to think about. Pyra gets out of your way so you can focus on building.
</Callout>

## ✨ Create

**Build with confidence and clarity.**

Development should be joyful, not frustrating. Pyra enhances your creative process through:

- **TypeScript-first design** - Full type safety without configuration
- **Clear error messages** - Helpful diagnostics that guide you to solutions
- **Hot module replacement** - See changes instantly without losing state
- **Framework flexibility** - Use any library or framework you prefer

### Developer Experience Matters

We believe that developer experience directly impacts product quality. When tools are fast and intuitive, developers can focus on solving real problems instead of fighting their toolchain.

## 🌊 Flow

**Stay in the zone, ship with confidence.**

Great tools maintain your flow state. Pyra keeps you productive by:

- **Predictable behavior** - Consistent results across development and production
- **Optimized builds** - Automatic code splitting and tree-shaking
- **Extensible architecture** - Powerful plugins when you need customization
- **Production-ready** - Deploy with confidence knowing your build is optimized

<Callout type="note">
  Flow state is when you're fully immersed in your work. Pyra is designed to preserve that state by minimizing interruptions and maximizing responsiveness.
</Callout>

## Design Principles

### 1. Speed is a Feature

Performance isn't just about benchmarks—it's about maintaining developer momentum. Every millisecond saved in HMR or build time compounds over a project's lifetime.

### 2. Simplicity Over Complexity

We favor simple, composable APIs over complex configuration. If a feature requires extensive setup, we reconsider whether it belongs in the core.

### 3. TypeScript as a First-Class Citizen

TypeScript isn't an afterthought—it's fundamental to Pyra's design. Type safety improves code quality and developer confidence.

### 4. Leverage Native Platform Features

We build on web standards and native browser capabilities rather than reinventing the wheel. ESM, import maps, and modern JavaScript features are core to Pyra's architecture.

### 5. Extensibility Through Plugins

The core should be lean and fast. Advanced features live in plugins, keeping the base tool simple while enabling powerful customization.

## Inspiration

Pyra.js draws inspiration from:

- **Vite** - For pioneering native ESM development
- **esbuild** - For proving native tooling can be orders of magnitude faster
- **Rollup** - For elegant plugin architecture
- **TypeScript** - For demonstrating the value of type safety

## Community Values

We're building Pyra.js in the open with these values:

- **Transparency** - Open development, clear roadmap, honest communication
- **Inclusivity** - Welcoming contributors of all skill levels
- **Pragmatism** - Solving real problems over chasing trends
- **Quality** - Shipping features when they're ready, not rushed

  Want to contribute? Check out our [GitHub repository](https://github.com/pyra-js/pyra) and join the conversation.

## The Name "Pyra"

The name "Pyra" comes from the Greek word for fire (πυρά). Like fire, Pyra:

- **Ignites** quickly and spreads rapidly
- **Transforms** raw materials into something useful
- **Provides energy** to power your development
- **Illuminates** the path forward with clear feedback

Fire has been humanity's tool for creation since the beginning. Pyra aims to be that tool for modern web development.
