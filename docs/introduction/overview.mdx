# Overview

**Pyra.js** is a next-generation build tool designed for modern web development. Built from the ground up with TypeScript and performance in mind, Pyra delivers instant server starts, lightning-fast HMR, and optimized production builds.

## What is Pyra?

Pyra is a build tool that combines the best ideas from Vite, esbuild, and Rollup into a unified, opinionated developer experience. It's designed to:

- **Start instantly** — No bundling during development
- **Update instantly** — HMR that stays fast as your app grows
- **Build optimally** — Production builds optimized with Rollup
- **Stay simple** — Minimal configuration, maximum productivity

<Callout type="tip">
Pyra is perfect for React, Vue, Svelte, and vanilla TypeScript projects. If you're building a modern web app, Pyra has you covered.
</Callout>

## Key Features

### ⚡ Lightning Fast

Pyra uses native ES modules during development, meaning no bundling is required. Your dev server starts in milliseconds, not seconds.

```typescript
// Server starts instantly, even with 1000+ modules
import { createServer } from 'pyra';

const server = await createServer();
await server.listen();
// Ready in ~50ms
```

### 🔥 Hot Module Replacement

HMR that actually works. Pyra's HMR implementation is precise, fast, and reliable — even in large applications.

### 📦 Optimized Builds

Production builds use Rollup under the hood, with automatic code splitting, tree-shaking, and asset optimization.

### 🎯 TypeScript First

Full TypeScript support out of the box. No configuration needed. Pyra understands your `tsconfig.json` and respects your path aliases.

### 🔌 Plugin Ecosystem

Extend Pyra with a powerful plugin API compatible with Rollup plugins. Create custom transformations, add framework support, or integrate with your favorite tools.

## What Pyra Is Not

Pyra is **not**:

- A framework (it's a build tool)
- A replacement for your bundler in all scenarios (use Webpack for complex legacy apps)
- Production-ready for server-side rendering (SSR support is experimental)

<Callout type="note">
Pyra is currently in **MVP** stage. APIs may change before v1.0. Use in production at your own risk.
</Callout>

## Browser Support

Pyra targets modern browsers with native ES module support:

- Chrome ≥ 87
- Firefox ≥ 78
- Safari ≥ 14
- Edge ≥ 88

For legacy browser support, use the `@pyra/plugin-legacy` plugin.
