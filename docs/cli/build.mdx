# pyra build

<p className="text-xl text-gray-400 leading-relaxed">
  Build your application for production with optimizations.
</p>

## Usage

<Terminal>pyra build</Terminal>

This creates an optimized production build in the `dist/` directory.

## Output

<Terminal>pyra build</Terminal>

```

✓ 34 modules transformed.
dist/index.html                   0.45 kB
dist/assets/index-a3b4c5d6.css    2.14 kB │ gzip: 1.02 kB
dist/assets/index-e7f8g9h0.js   143.42 kB │ gzip: 46.33 kB
✓ built in 1.23s
```

## Options

### `--outDir <dir>`

Specify output directory:

<Terminal>pyra build --outDir build</Terminal>

### `--mode <mode>`

Set build mode:

<Terminal>pyra build --mode staging</Terminal>

### `--base <path>`

Set base public path:

<Terminal>pyra build --base /my-app/</Terminal>

### `--sourcemap`

Generate sourcemaps:

<Terminal>pyra build --sourcemap</Terminal>

### `--minify [minifier]`

Enable/disable minification:

<Terminal>pyra build --minify false</Terminal>

<Terminal>pyra build --minify terser</Terminal>

### `--watch`

Watch mode for continuous builds:

<Terminal>pyra build --watch</Terminal>

<Callout type="tip">
  Watch mode is useful for library development or when integrating with other build tools.
</Callout>

## Configuration

Configure build in `pyra.config.ts`:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';

export default defineConfig({
  build: {
    // Output directory
    outDir: 'dist',
    
    // Assets directory (relative to outDir)
    assetsDir: 'assets',
    
    // Generate sourcemaps
    sourcemap: true,
    
    // Minification
    minify: 'esbuild',
    
    // Browser target
    target: 'es2020',
    
    // Chunk size warning limit (KB)
    chunkSizeWarningLimit: 500,
    
    // Rollup options
    rollupOptions: {
      output: {
        // Manual chunks
        manualChunks: {
          vendor: ['react', 'react-dom'],
          utils: ['lodash', 'date-fns']
        },
        
        // Asset file names
        assetFileNames: 'assets/[name]-[hash][extname]',
        
        // Chunk file names
        chunkFileNames: 'assets/[name]-[hash].js',
        
        // Entry file names
        entryFileNames: 'assets/[name]-[hash].js'
      }
    }
  }
});`}
</GradientCode>

## Code Splitting

### Automatic Splitting

Pyra automatically splits code:

<GradientCode language="typescript">
{`// Dynamic imports create separate chunks
const Dashboard = lazy(() => import('./Dashboard'));

// Vendor dependencies are split automatically
import React from 'react';
import lodash from 'lodash';`}
</GradientCode>

### Manual Chunks

Control chunk splitting:

<GradientCode language="typescript" title="pyra.config.ts">
{`export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks(id) {
          // Vendor chunk
          if (id.includes('node_modules')) {
            return 'vendor';
          }
          
          // UI components chunk
          if (id.includes('/src/components/')) {
            return 'components';
          }
          
          // Utils chunk
          if (id.includes('/src/utils/')) {
            return 'utils';
          }
        }
      }
    }
  }
});`}
</GradientCode>

## Minification

### esbuild (default)

Fast minification with esbuild:

<GradientCode language="typescript">
{`export default defineConfig({
  build: {
    minify: 'esbuild'
  }
});`}
</GradientCode>

### Terser

More aggressive minification:

<GradientCode language="typescript">
{`export default defineConfig({
  build: {
    minify: 'terser',
    terserOptions: {
      compress: {
        drop_console: true,
        drop_debugger: true
      }
    }
  }
});`}
</GradientCode>

<Callout type="note">
  Terser is slower but produces smaller bundles. Use it when bundle size is critical.
</Callout>

## Library Mode

Build as a library:

<GradientCode language="typescript" title="pyra.config.ts">
{`export default defineConfig({
  build: {
    lib: {
      entry: './src/index.ts',
      name: 'MyLib',
      fileName: (format) => \`my-lib.\${format}.js\`,
      formats: ['es', 'cjs', 'umd']
    },
    rollupOptions: {
      // Externalize dependencies
      external: ['react', 'react-dom'],
      output: {
        globals: {
          react: 'React',
          'react-dom': 'ReactDOM'
        }
      }
    }
  }
});`}
</GradientCode>

## Optimization Tips

### 1. Code Splitting

Split large dependencies:

<GradientCode language="typescript">
{`// Instead of
import { format } from 'date-fns';

// Use
import format from 'date-fns/format';`}
</GradientCode>

### 2. Tree Shaking

Ensure tree shaking works:

<GradientCode language="typescript">
{`// ✅ Named exports (tree-shakeable)
export const foo = () => {};
export const bar = () => {};

// ❌ Default export (not tree-shakeable)
export default { foo, bar };`}
</GradientCode>

### 3. Dynamic Imports

Lazy load routes and components:

<GradientCode language="typescript">
{`const Dashboard = lazy(() => import('./Dashboard'));
const Settings = lazy(() => import('./Settings'));`}
</GradientCode>

### 4. Remove Console Logs

<GradientCode language="typescript">
{`export default defineConfig({
  build: {
    minify: 'terser',
    terserOptions: {
      compress: {
        drop_console: true
      }
    }
  }
});`}
</GradientCode>

## Troubleshooting

### Large Bundle Size

1. Analyze with `--report`
2. Check for duplicate dependencies
3. Use dynamic imports
4. Enable tree shaking

### Build Errors

Check:
- TypeScript errors: `tsc --noEmit`
- Missing dependencies
- Circular dependencies
- Invalid imports

### Slow Builds

Optimize:
- Reduce number of chunks
- Disable sourcemaps
- Use esbuild instead of terser
- Exclude large dependencies from pre-bundling
