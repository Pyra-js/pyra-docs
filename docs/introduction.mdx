# 🔺 Introduction to Pyra

## A modern developer engine designed to stay out of your way.

**Pyra** is built for people who want clarity over magic, speed without ceremony, and control without complexity. It provides a clean, understandable foundation for building applications, frameworks, and tools, without forcing you into a maze of configuration or unclear internals.

Think of Pyra as the **engine** beneath your project: fast to start, predictable to extend, and transparent in how it works.

---

## ⚡ What is Pyra?

Pyra is a **modular build and runtime platform** for modern JavaScript and TypeScript projects.

It began as a fast build tool, but its goal is broader: to provide a cohesive system for development, building, and running applications with minimal friction. Pyra favors conventions over wrappers, native standards over custom abstractions, and explicit behavior over hidden magic.

At its core, Pyra is guided by three principles:

* Simplicity by default. You shouldn’t need to understand the tool to start using it. Sensible defaults get you running immediately, and configuration is only introduced when you actually need it.

* Performance as a baseline. Fast startup, fast rebuilds, and efficient output are not optional optimizations, they’re the default behavior.

* Extensibility without lock-in. Pyra is designed to be extended through clear contracts: plugins, adapters, and CLI hooks that fit into your workflow without fighting it.

---

## 🔥 Why Pyra Exists

Modern tooling often forces you to choose between too small and too heavy.

Some tools are incredibly fast but limited in scope. Others are powerful but difficult to reason about, customize, or debug once something goes wrong. Pyra exists to sit deliberately between those extremes.

You might use Pyra if you’ve ever said:

> “I want a build system I can actually understand.”  
> “I like modern tooling, but I don’t want framework-specific magic everywhere”  
> “I want to build something on top of my tooling, not fight it.”

Pyra is built for framework authors, library developers, and indie builders who want a foundation that scales with ambition, not complexity. It’s a system you can start small with, and grow into, without needing to rewrite everything once your project matures.
