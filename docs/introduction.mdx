# 🔺 Introduction to Pyra

Welcome to **Pyra**, a blazing-fast build tool and modern developer engine designed for simplicity, speed, and extensibility. Think of Pyra as the **spark beneath your stack**, a toolchain that ignites your project with minimal setup, intuitive configuration, and a deep focus on developer experience.

---

## ⚡ What is Pyra?

Pyra is a **modular build system** for modern JavaScript and TypeScript projects. It takes inspiration from the elegance of **Vite**-like speed with the flexibility of **plugin-driven architecture**, allowing developers to craft, extend, and optimize builds for any kind of project, from small scripts to full-scale frameworks.

At its core, Pyra is powered by three guiding principles:

1. **Simplicity over configuration** – You shouldn’t need 50 lines of config to start coding.  

2. **Performance as a baseline** – Everything is fast by default, no tuning required.  

3. **Extensibility by design** – Extend Pyra with plugins, custom loaders, and CLI hooks that fit your workflow.

---

## 🔥 Why Pyra Exists

Most modern bundlers are either too minimal or too massive.   Pyra was built to **bridge that gap**, providing a developer-first environment that’s _simple enough for beginners_ yet _powerful enough for toolsmiths_.

You might use Pyra if you’ve ever said:

> “I wish Vite’s plugin system was more modular.”  
> “I like Rollup’s output but not its setup.”  
> “I want to build my own framework or CLI but don’t want to reinvent bundling.”

Pyra aims to be **the foundation for framework authors, library developers, and indie makers**, a build engine you can actually understand, customize, and ship with confidence.
