# Configuration Overview

<p className="text-xl text-gray-400 leading-relaxed">
  Pyra.js works with zero configuration, but offers powerful customization when you need it.
</p>

## Config File

Create a `pyra.config.ts` file in your project root:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';

export default defineConfig({
  // Your configuration here
});`}
</GradientCode>

<Callout type="tip">
  Use `defineConfig` for full TypeScript IntelliSense and type checking.
</Callout>

## Basic Configuration

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';
import react from '@pyra/plugin-react';

export default defineConfig({
  // Plugins
  plugins: [react()],
  
  // Development server
  server: {
    port: 3000,
    open: true,
    host: true
  },
  
  // Build options
  build: {
    outDir: 'dist',
    sourcemap: true,
    minify: 'esbuild'
  },
  
  // Module resolution
  resolve: {
    alias: {
      '@': '/src',
      '@components': '/src/components'
    }
  }
});`}
</GradientCode>

## Configuration Options

### Root Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `root` | `string` | `process.cwd()` | Project root directory |
| `base` | `string` | `'/'` | Base public path |
| `mode` | `string` | `'development'` | App mode |
| `plugins` | `Plugin[]` | `[]` | Array of plugins |

### Server Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `server.port` | `number` | `3000` | Dev server port |
| `server.host` | `boolean \| string` | `false` | Expose to network |
| `server.open` | `boolean \| string` | `false` | Open browser on start |
| `server.cors` | `boolean \| CorsOptions` | `true` | Enable CORS |
| `server.hmr` | `boolean \| HmrOptions` | `true` | Enable HMR |

### Build Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `build.outDir` | `string` | `'dist'` | Output directory |
| `build.sourcemap` | `boolean \| 'inline'` | `false` | Generate sourcemaps |
| `build.minify` | `boolean \| 'esbuild'` | `'esbuild'` | Minification |
| `build.target` | `string` | `'es2020'` | Browser target |
| `build.chunkSizeWarningLimit` | `number` | `500` | Chunk size warning (KB) |

<Callout type="note">
  See the [Server Options](/docs/configuration/server-options), [Build Options](/docs/configuration/build-options), and [Resolve Options](/docs/configuration/resolve-options) pages for detailed documentation.
</Callout>

## Conditional Configuration

Use functions for environment-specific config:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';

export default defineConfig(({ command, mode }) => {
  if (command === 'serve') {
    // Dev-specific config
    return {
      server: {
        port: 3000
      }
    };
  } else {
    // Build-specific config
    return {
      build: {
        minify: true,
        sourcemap: false
      }
    };
  }
});`}
</GradientCode>

## Environment Variables in Config

Access environment variables:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig, loadEnv } from 'pyra';

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), '');
  
  return {
    define: {
      __APP_VERSION__: JSON.stringify(env.APP_VERSION)
    }
  };
});`}
</GradientCode>

## Async Configuration

Return a promise for async setup:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';

export default defineConfig(async ({ command, mode }) => {
  const data = await fetchSomeConfig();
  
  return {
    // Use data in config
    define: {
      __CONFIG__: JSON.stringify(data)
    }
  };
});`}
</GradientCode>

## Multiple Configs

Export an array for multiple builds:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';

export default defineConfig([
  // Client build
  {
    build: {
      outDir: 'dist/client'
    }
  },
  // SSR build
  {
    build: {
      ssr: true,
      outDir: 'dist/server'
    }
  }
]);`}
</GradientCode>

## Config IntelliSense

For JavaScript configs, use JSDoc:

<GradientCode language="javascript" title="pyra.config.js">
{`/** @type {import('pyra').UserConfig} */
export default {
  server: {
    port: 3000
  }
};`}
</GradientCode>

Or use `defineConfig`:

<GradientCode language="javascript" title="pyra.config.js">
{`import { defineConfig } from 'pyra';

export default defineConfig({
  server: {
    port: 3000
  }
});`}
</GradientCode>

## Config File Locations

Pyra looks for config files in this order:

1. `pyra.config.ts`
2. `pyra.config.js`
3. `pyra.config.mjs`

Specify a custom config file:

<GradientCode language="bash">
{`pyra --config my-config.ts`}
</GradientCode>

## Shared Config

Extract shared config to a separate file:

<GradientCode language="typescript" title="config/shared.ts">
{`export const sharedConfig = {
  server: {
    port: 3000
  }
};`}
</GradientCode>

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';
import { sharedConfig } from './config/shared';

export default defineConfig({
  ...sharedConfig,
  // Additional config
});`}
</GradientCode>
