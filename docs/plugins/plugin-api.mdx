# Plugin API

Pyra's plugin system is built on Rollup's plugin interface with additional hooks for dev server and HMR functionality.

## Creating a Plugin

A Pyra plugin is an object with a `name` property and one or more hooks:

<GradientCode language="typescript">
import type { PyraPlugin } from 'pyra';

export default function myPlugin(): PyraPlugin {
  return {
    name: 'my-plugin',
    
    // Plugin hooks go here
    transform(code, id) {
      if (id.endsWith('.custom')) {
        return {
          code: transformCode(code),
          map: null,
        };
      }
    },
  };
}
</GradientCode>

## Universal Hooks

These hooks work in both dev and build modes.

### `config`

Modify Pyra config before it's resolved. Return partial config to merge.

<GradientCode language="typescript">
config(config, env) {
  if (env.command === 'serve') {
    return {
      server: {
        port: 3000,
      },
    };
  }
}
</GradientCode>

**Parameters:**
- `config`: Current user config
- `env`: `{ command: 'serve' | 'build', mode: string }`

**Returns:** Partial config or `null`

### `configResolved`

Called after config is resolved. Use this to read the final config.

<GradientCode language="typescript">
configResolved(resolvedConfig) {
  console.log('Final config:', resolvedConfig);
}
</GradientCode>

<Callout type="note">
This hook is called after all `config` hooks have run and the config is finalized.
</Callout>

### `resolveId`

Custom module resolution. Return a module ID to resolve it.

<GradientCode language="typescript">
resolveId(source, importer) {
  if (source === 'virtual-module') {
    return '\0virtual-module';
  }
}
</GradientCode>

**Parameters:**
- `source`: The module being imported
- `importer`: The file doing the importing

**Returns:** Resolved ID or `null`

<Callout type="tip">
Prefix virtual module IDs with `\0` to prevent other plugins from processing them.
</Callout>

### `load`

Load module content. Return code for the module.

<GradientCode language="typescript">
load(id) {
  if (id === '\0virtual-module') {
    return 'export default "Hello from virtual module"';
  }
}
</GradientCode>

**Parameters:**
- `id`: The module ID to load

**Returns:** Module code or `null`

### `transform`

Transform module code. This is where most processing happens.

<GradientCode language="typescript">
transform(code, id) {
  if (id.endsWith('.md')) {
    return {
      code: `export default ${JSON.stringify(markdownToHtml(code))}`,
      map: null,
    };
  }
}
</GradientCode>

**Parameters:**
- `code`: Module source code
- `id`: Module ID

**Returns:** `{ code: string, map?: SourceMap | null }`

## Dev Server Hooks

These hooks only run during development.

### `configureServer`

Configure the dev server. Add middleware, set up WebSocket handlers, etc.

<GradientCode language="typescript">
configureServer(server) {
  server.middlewares.use((req, res, next) => {
    if (req.url === '/api/health') {
      res.end('OK');
    } else {
      next();
    }
  });
}
</GradientCode>

**Parameters:**
- `server`: Pyra dev server instance

**Returns:** Post-middleware hook function (optional)

### `transformIndexHtml`

Transform `index.html`. Inject tags, modify content, etc.

<GradientCode language="typescript">
transformIndexHtml(html) {
  return html.replace(
    '</head>',
    '<script src="/analytics.js"></script></head>'
  );
}
</GradientCode>

Or return tag descriptors:

<GradientCode language="typescript">
transformIndexHtml() {
  return [
    {
      tag: 'script',
      attrs: { src: '/analytics.js' },
      injectTo: 'head',
    },
  ];
}
</GradientCode>

### `handleHotUpdate`

Custom HMR handling. Control which modules reload.

<GradientCode language="typescript">
handleHotUpdate({ file, server }) {
  if (file.endsWith('.custom')) {
    console.log('Custom file changed:', file);
    
    // Trigger full reload
    server.ws.send({
      type: 'full-reload',
      path: '*',
    });
    
    // Return empty array to prevent default HMR
    return [];
  }
}
</GradientCode>

**Parameters:**
- `file`: Changed file path
- `server`: Dev server instance
- `modules`: Affected modules
- `timestamp`: Update timestamp

**Returns:** Array of modules to update, or `void`

## Build Hooks

These hooks only run during production builds.

### `buildStart`

Called when build starts.

<GradientCode language="typescript">
buildStart(options) {
  console.log('Build starting with options:', options);
}
</GradientCode>

### `buildEnd`

Called when build ends (success or failure).

<GradientCode language="typescript">
buildEnd(error) {
  if (error) {
    console.error('Build failed:', error);
  } else {
    console.log('Build succeeded!');
  }
}
</GradientCode>

### `generateBundle`

Called when bundle is generated. Modify output files.

<GradientCode language="typescript">
generateBundle(options, bundle) {
  // Add a custom file to the bundle
  this.emitFile({
    type: 'asset',
    fileName: 'custom.txt',
    source: 'Custom content',
  });
}
</GradientCode>

## Plugin Ordering

Control when your plugin runs with the `enforce` option:

<GradientCode language="typescript">
export default function myPlugin(): PyraPlugin {
  return {
    name: 'my-plugin',
    enforce: 'pre', // 'pre' | 'post' | undefined
    
    // ... hooks
  };
}
</GradientCode>

- **`pre`**: Run before Pyra core plugins
- **`post`**: Run after Pyra core plugins
- **`undefined`**: Run in normal order

## Conditional Application

Apply plugins only in certain modes:

<GradientCode language="typescript">
export default function myPlugin(): PyraPlugin {
  return {
    name: 'my-plugin',
    apply: 'build', // 'serve' | 'build' | function
    
    // ... hooks
  };
}
</GradientCode>

Or use a function:

<GradientCode language="typescript">
apply(config, { command }) {
  return command === 'build' && config.build.ssr;
}
</GradientCode>

## Complete Example

Here's a complete plugin that transforms `.env` files:

<GradientCode language="typescript">
import type { PyraPlugin } from 'pyra';
import fs from 'fs';
import path from 'path';

export default function envPlugin(): PyraPlugin {
  let config: any;
  
  return {
    name: 'pyra-plugin-env',
    enforce: 'pre',
    
    configResolved(resolvedConfig) {
      config = resolvedConfig;
    },
    
    resolveId(source) {
      if (source === 'virtual:env') {
        return '\0virtual:env';
      }
    },
    
    load(id) {
      if (id === '\0virtual:env') {
        const envPath = path.resolve(config.root, '.env');
        
        if (!fs.existsSync(envPath)) {
          return 'export default {}';
        }
        
        const envContent = fs.readFileSync(envPath, 'utf-8');
        const env = {};
        
        envContent.split('\n').forEach(line => {
          const [key, value] = line.split('=');
          if (key && value) {
            env[key.trim()] = value.trim();
          }
        });
        
        return `export default ${JSON.stringify(env)}`;
      }
    },
    
    handleHotUpdate({ file, server }) {
      if (file.endsWith('.env')) {
        // Reload when .env changes
        server.ws.send({
          type: 'full-reload',
          path: '*',
        });
      }
    },
  };
}
</GradientCode>

Usage:

<GradientCode language="typescript">
// pyra.config.ts
import { defineConfig } from 'pyra';
import envPlugin from './plugins/env-plugin';

export default defineConfig({
  plugins: [envPlugin()],
});
</GradientCode>

<GradientCode language="typescript">
// In your app
import env from 'virtual:env';

console.log(env.API_KEY);
</GradientCode>

## Plugin Context

Inside hooks, `this` provides useful methods:

### `this.emitFile()`

Emit a file to the output directory.

<GradientCode language="typescript">
this.emitFile({
  type: 'asset',
  fileName: 'custom.json',
  source: JSON.stringify(data),
});
</GradientCode>

### `this.error()`

Throw a build error.

<GradientCode language="typescript">
if (!isValid) {
  this.error('Invalid configuration');
}
</GradientCode>

### `this.warn()`

Emit a warning.

<GradientCode language="typescript">
this.warn('Deprecated API used');
</GradientCode>

## TypeScript Support

Get full type safety for your plugins:

<GradientCode language="typescript">
import type { PyraPlugin, ResolvedConfig } from 'pyra';

export default function myPlugin(): PyraPlugin {
  let config: ResolvedConfig;
  
  return {
    name: 'my-plugin',
    
    configResolved(resolvedConfig) {
      config = resolvedConfig;
    },
    
    transform(code: string, id: string) {
      // TypeScript knows the types!
      return {
        code: code.toUpperCase(),
        map: null,
      };
    },
  };
}
</GradientCode>

## Best Practices

<Step number={1}>
**Use descriptive names** — Prefix with `pyra-plugin-` for npm packages
</Step>

<Step number={2}>
**Handle errors gracefully** — Use `this.error()` and `this.warn()`
</Step>

<Step number={3}>
**Cache expensive operations** — Store results in `configResolved`
</Step>

<Step number={4}>
**Document your plugin** — Include usage examples and TypeScript types
</Step>

<Step number={5}>
**Test in both modes** — Ensure your plugin works in dev and build
</Step>

<Callout type="warn">
Avoid heavy synchronous operations in `transform` hooks. They run on every file and can slow down builds.
</Callout>
