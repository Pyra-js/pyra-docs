# Plugins Overview

<p className="text-xl text-gray-400 leading-relaxed">
  Extend Pyra.js functionality with a powerful, flexible plugin system.
</p>

## What are Plugins?

Plugins are the primary way to extend Pyra.js. They can:

- Transform source code
- Resolve module imports
- Add custom dev server middleware
- Modify the build output
- Inject code at various lifecycle hooks

## Using Plugins

Install and configure plugins in `pyra.config.ts`:

<GradientCode language="typescript" title="pyra.config.ts">
{`import { defineConfig } from 'pyra';
import react from '@pyra/plugin-react';
import tailwind from '@pyra/plugin-tailwind';

export default defineConfig({
  plugins: [
    react(),
    tailwind({
      config: './tailwind.config.js'
    })
  ]
});`}
</GradientCode>

## Official Plugins

### @pyra/plugin-react <Badge variant="mvp">MVP</Badge>

React support with Fast Refresh:

<GradientCode language="typescript">
{`import react from '@pyra/plugin-react';

export default defineConfig({
  plugins: [
    react({
      // Enable Fast Refresh
      fastRefresh: true,
      
      // Babel options
      babel: {
        plugins: ['babel-plugin-styled-components']
      }
    })
  ]
});`}
</GradientCode>

### @pyra/plugin-vue <Badge variant="mvp">MVP</Badge>

Vue 3 support with SFC compilation:

<GradientCode language="typescript">
{`import vue from '@pyra/plugin-vue';

export default defineConfig({
  plugins: [
    vue({
      // Template options
      template: {
        compilerOptions: {
          isCustomElement: (tag) => tag.startsWith('ion-')
        }
      }
    })
  ]
});`}
</GradientCode>

### @pyra/plugin-svelte <Badge variant="beta">Beta</Badge>

Svelte support with HMR:

<GradientCode language="typescript">
{`import svelte from '@pyra/plugin-svelte';

export default defineConfig({
  plugins: [
    svelte({
      // Svelte compiler options
      compilerOptions: {
        dev: true
      }
    })
  ]
});`}
</GradientCode>

## Plugin Ordering

Plugins execute in the order they're defined:

<GradientCode language="typescript">
{`export default defineConfig({
  plugins: [
    // Runs first
    pluginA(),
    
    // Runs second
    pluginB(),
    
    // Runs last
    pluginC()
  ]
});`}
</GradientCode>

<Callout type="tip">
  Some plugins need to run before or after others. Check plugin documentation for ordering requirements.
</Callout>

## Conditional Plugins

Apply plugins conditionally:

<GradientCode language="typescript">
{`export default defineConfig(({ command, mode }) => ({
  plugins: [
    react(),
    
    // Only in production
    mode === 'production' && compression(),
    
    // Only in development
    command === 'serve' && devTools()
  ].filter(Boolean)
}));`}
</GradientCode>

## Plugin Options

Most plugins accept configuration options:

<GradientCode language="typescript">
{`export default defineConfig({
  plugins: [
    myPlugin({
      // Plugin-specific options
      option1: 'value',
      option2: true,
      
      // Nested configuration
      advanced: {
        feature: 'enabled'
      }
    })
  ]
});`}
</GradientCode>

## Community Plugins

Browse community plugins on npm:

- Search for packages with `pyra-plugin-` prefix
- Check the [Awesome Pyra](https://github.com/pyra-js/awesome-pyra) list
- Share your own plugins!

<Callout type="note">
  Community plugins are maintained by third parties. Always review code and check maintenance status before using.
</Callout>

## Plugin Compatibility

Pyra plugins are compatible with Rollup plugins in most cases:

<GradientCode language="typescript">
{`import { defineConfig } from 'pyra';
import rollupPlugin from 'rollup-plugin-example';

export default defineConfig({
  plugins: [
    // Rollup plugin works in Pyra
    rollupPlugin()
  ]
});`}
</GradientCode>

<Callout type="warn">
  Some Rollup plugins may not work in dev mode since Pyra uses native ESM. They'll still work in production builds.
</Callout>
