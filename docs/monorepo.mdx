# 🧭 Monorepos and Workspaces 

A **monorepo** is a single repository that contains multiple projects, such as apps, libraries, and shared utilities, managed together under one version control system.  

**Workspaces** are the package-manager feature (from pnpm, npm, or yarn) that make this structure work seamlessly by linking local packages together.  

Pyra was designed with monorepos and workspaces in mind. It plays nicely with tools like pnpm, Turbo, and Nx, and makes it easy to share configs, dependencies, and utilities between apps without friction.
