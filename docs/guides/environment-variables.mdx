# Environment Variables

Pyra has built-in support for environment variables, making it easy to configure your app for different environments.

## .env Files

Pyra automatically loads environment variables from `.env` files in your project root:

```
.env                # loaded in all cases
.env.local          # loaded in all cases, ignored by git
.env.[mode]         # only loaded in specified mode
.env.[mode].local   # only loaded in specified mode, ignored by git
```

<Callout type="tip">
Add `.env.local` and `.env.*.local` to your `.gitignore` to keep secrets out of version control.
</Callout>

## File Priority

Files are loaded in this order (later files override earlier ones):

<Step number={1}>
`.env` — Base environment variables
</Step>

<Step number={2}>
`.env.local` — Local overrides (not committed)
</Step>

<Step number={3}>
`.env.[mode]` — Mode-specific variables (e.g., `.env.production`)
</Step>

<Step number={4}>
`.env.[mode].local` — Mode-specific local overrides
</Step>

## Example .env Files

### .env

<GradientCode language="bash">
# Base configuration
PYRA_APP_TITLE=My Pyra App
PYRA_API_URL=https://api.example.com
PYRA_ENABLE_ANALYTICS=false
</GradientCode>

### .env.development

<GradientCode language="bash">
# Development overrides
PYRA_API_URL=http://localhost:8080
PYRA_ENABLE_ANALYTICS=false
PYRA_DEBUG=true
</GradientCode>

### .env.production

<GradientCode language="bash">
# Production overrides
PYRA_API_URL=https://api.production.com
PYRA_ENABLE_ANALYTICS=true
PYRA_DEBUG=false
</GradientCode>

### .env.local

<GradientCode language="bash">
# Local secrets (not committed)
PYRA_API_KEY=sk_live_abc123xyz789
PYRA_STRIPE_KEY=pk_test_def456uvw012
</GradientCode>

## Accessing Environment Variables

### In Client Code

Only variables prefixed with `PYRA_` are exposed to your client code:

<GradientCode language="typescript">
// ✅ Accessible (prefixed with PYRA_)
const apiUrl = import.meta.env.PYRA_API_URL;
const appTitle = import.meta.env.PYRA_APP_TITLE;

// ❌ Not accessible (no PYRA_ prefix)
const secret = import.meta.env.SECRET_KEY; // undefined
</GradientCode>

<Callout type="warn">
Never put secrets in `PYRA_` prefixed variables! They're exposed to the client and visible in your bundle.
</Callout>

### Built-in Variables

Pyra provides these variables automatically:

<GradientCode language="typescript">
import.meta.env.MODE          // 'development' | 'production'
import.meta.env.BASE_URL      // base public path
import.meta.env.PROD          // boolean, true in production
import.meta.env.DEV           // boolean, true in development
import.meta.env.SSR           // boolean, true during SSR
</GradientCode>

### TypeScript IntelliSense

Add type definitions for your environment variables:

<GradientCode language="typescript">
// src/env.d.ts
/// <reference types="pyra/client" />

interface ImportMetaEnv {
  readonly PYRA_API_URL: string;
  readonly PYRA_APP_TITLE: string;
  readonly PYRA_ENABLE_ANALYTICS: string;
  readonly PYRA_DEBUG?: string;
}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}
</GradientCode>

Now you get autocomplete and type checking:

<GradientCode language="typescript">
// TypeScript knows this exists and is a string
const apiUrl: string = import.meta.env.PYRA_API_URL;
</GradientCode>

## Custom Prefix

Change the prefix from `PYRA_` to something else:

<GradientCode language="typescript">
// pyra.config.ts
import { defineConfig } from 'pyra';

export default defineConfig({
  envPrefix: 'APP_', // Now use APP_ instead of PYRA_
});
</GradientCode>

Or use multiple prefixes:

<GradientCode language="typescript">
export default defineConfig({
  envPrefix: ['PYRA_', 'APP_', 'PUBLIC_'],
});
</GradientCode>

## Modes

Pyra runs in different modes depending on the command:

- **`development`** — `pyra dev`
- **`production`** — `pyra build`

Override the mode with `--mode`:

<Terminal>
pyra build --mode staging
</Terminal>

This loads `.env.staging` and `.env.staging.local`.

## Using in HTML

Environment variables work in `index.html` too:

<GradientCode language="html">
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>%PYRA_APP_TITLE%</title>
    <meta name="description" content="%PYRA_APP_DESCRIPTION%" />
  </head>
  <body>
    <div id="app"></div>
    <script type="module" src="/src/main.ts"></script>
  </body>
</html>
</GradientCode>

<Callout type="note">
HTML replacement happens at build time. The variables must be defined when you run `pyra build`.
</Callout>

## Define Global Constants

For non-environment variables, use `define`:

<GradientCode language="typescript">
// pyra.config.ts
import { defineConfig } from 'pyra';

export default defineConfig({
  define: {
    __APP_VERSION__: JSON.stringify('1.0.0'),
    __BUILD_TIME__: JSON.stringify(new Date().toISOString()),
    __FEATURE_FLAGS__: JSON.stringify({
      newUI: true,
      betaFeatures: false,
    }),
  },
});
</GradientCode>

Access them directly:

<GradientCode language="typescript">
console.log('App version:', __APP_VERSION__);
console.log('Built at:', __BUILD_TIME__);

if (__FEATURE_FLAGS__.newUI) {
  // Use new UI
}
</GradientCode>

<Callout type="tip">
Remember to add type definitions for your global constants in `env.d.ts`.
</Callout>

## Best Practices

### 1. Use .env.local for Secrets

<GradientCode language="bash">
# .env.local (not committed)
PYRA_STRIPE_SECRET=sk_live_...
PYRA_DATABASE_URL=postgres://...
</GradientCode>

### 2. Provide Defaults in .env

<GradientCode language="bash">
# .env (committed)
PYRA_API_URL=https://api.example.com
PYRA_TIMEOUT=5000
</GradientCode>

### 3. Document Required Variables

Create a `.env.example` file:

<GradientCode language="bash">
# .env.example
PYRA_API_URL=https://api.example.com
PYRA_API_KEY=your_api_key_here
PYRA_ENABLE_ANALYTICS=false
</GradientCode>

### 4. Validate at Runtime

<GradientCode language="typescript">
// src/config.ts
const requiredEnvVars = [
  'PYRA_API_URL',
  'PYRA_APP_TITLE',
] as const;

for (const envVar of requiredEnvVars) {
  if (!import.meta.env[envVar]) {
    throw new Error(`Missing required environment variable: ${envVar}`);
  }
}

export const config = {
  apiUrl: import.meta.env.PYRA_API_URL,
  appTitle: import.meta.env.PYRA_APP_TITLE,
  enableAnalytics: import.meta.env.PYRA_ENABLE_ANALYTICS === 'true',
};
</GradientCode>

### 5. Use Type-Safe Config

<GradientCode language="typescript">
// src/config.ts
interface Config {
  apiUrl: string;
  appTitle: string;
  enableAnalytics: boolean;
  debug: boolean;
}

function getConfig(): Config {
  return {
    apiUrl: import.meta.env.PYRA_API_URL,
    appTitle: import.meta.env.PYRA_APP_TITLE,
    enableAnalytics: import.meta.env.PYRA_ENABLE_ANALYTICS === 'true',
    debug: import.meta.env.PYRA_DEBUG === 'true',
  };
}

export const config = getConfig();
</GradientCode>

## Common Patterns

### API Configuration

<GradientCode language="typescript">
// src/api/client.ts
const API_URL = import.meta.env.PYRA_API_URL;
const API_KEY = import.meta.env.PYRA_API_KEY;

export async function fetchData(endpoint: string) {
  const response = await fetch(`${API_URL}${endpoint}`, {
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
    },
  });
  
  return response.json();
}
</GradientCode>

### Feature Flags

<GradientCode language="typescript">
// src/features.ts
export const features = {
  newUI: import.meta.env.PYRA_FEATURE_NEW_UI === 'true',
  betaFeatures: import.meta.env.PYRA_FEATURE_BETA === 'true',
  analytics: import.meta.env.PYRA_ENABLE_ANALYTICS === 'true',
};

// Usage
if (features.newUI) {
  // Render new UI
}
</GradientCode>

### Environment-Specific Behavior

<GradientCode language="typescript">
// src/logger.ts
const isDev = import.meta.env.DEV;

export function log(...args: any[]) {
  if (isDev) {
    console.log('[DEV]', ...args);
  }
}

export function error(...args: any[]) {
  if (isDev) {
    console.error('[DEV]', ...args);
  } else {
    // Send to error tracking service
    sendToSentry(args);
  }
}
</GradientCode>

## Troubleshooting

### Variables Not Loading

<Callout type="warn">
Make sure your `.env` file is in the project root (same directory as `pyra.config.ts`).
</Callout>

### Variables Undefined

Check that:
1. Variable is prefixed with `PYRA_` (or your custom prefix)
2. `.env` file is in the correct location
3. You've restarted the dev server after changing `.env`

### TypeScript Errors

Add type definitions in `src/env.d.ts`:

<GradientCode language="typescript">
interface ImportMetaEnv {
  readonly PYRA_YOUR_VARIABLE: string;
}
</GradientCode>
